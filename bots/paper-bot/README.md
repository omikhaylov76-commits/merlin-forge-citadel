# paper-bot — эталонный картридж Контракта Бота v0

Фейковый бот: соблюдает [Контракт Бота v0](../../wiki/concepts/bot-contract.md), к бирже НЕ ходит
(paper-only, без ключей). Первый картридж платформы — по нему лепится обёртка Пифагора и будущие
движки. Обкатывает конвейер целиком до подключения реального движка. Единственная рантайм-зависимость —
`httpx`.

## Как картридж реализует Контракт (шов S4)

**Вход (env при старте).** `MF_INSTANCE_ID`, `MF_INSTANCE_TOKEN` (скоуп — свой инстанс), `MF_CORE_URL`.
Опц.: `MF_SEED` (детерминизм сделок), `MF_TICK_S`, `MF_HEARTBEAT_S`, `MF_POLL_WAIT_S`. Биржевые
переменные в paper игнорируются. Токен приходит один раз — в БД ядра только его хэш.

**Наружу (push телеметрии, токеном инстанса).**
- `heartbeat` — каждый оборот (≤60с): {status, uptime_s, contract_version=v0}. Кормит stale-скан
  часового; молчание >3×60с → health stale/dead (MFC-003).
- `equity` — точка кривой: детерминированный синус вокруг базы + вклад открытой позиции, currency=USDT.
- `trades` — «сделки» из seeded-ГПСЧ (вход/закрытие); dedup по exec_id.
- `events` — например `kill_switch` при stop_close.

**Команды (long-poll, честные семантики ADR-0005).** Картридж опрашивает `GET /commands/next` и
исполняет ЧЕСТНО:
- `pause` — стоп НОВЫХ входов; открытые позиции ДЕРЖАТСЯ (не закрываются). ack ok.
- `resume` — возобновляет входы. ack ok.
- `stop_close` — ЗАКРЫВАЕТ позицию (эмитит закрывающую сделку + event `kill_switch`) и ВСТАЁТ
  (процесс завершается). ack ok. Если закрыть не удалось — ack error (ядро держит липкость, не гасит).

## Структура
- `app/config.py` — env Контракта. · `app/engine.py` — PaperEngine: детерминированный движок (синус,
  позиция, seeded-сделки, семантики pause/stop_close). · `app/client.py` — httpx-клиент API S4
  (push + poll/ack). · `app/bot.py` — цикл (heartbeat + tick-телеметрия + команды). · `app/main.py` — запуск.

## Запуск / тесты
`pip install -e ".[dev]"` в `bots/paper-bot/`. Тесты БД/сети не требуют (детерминизм + httpx.MockTransport):
`PYTHONPATH="$PWD" pytest -q`. Боевой запуск (env из деплоя): `python -m app.main`.
