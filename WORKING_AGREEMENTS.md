# Рабочие соглашения Merlin Forge Citadel

§1. Роли: Оператор (владелец) · Куратор (Cowork-сессия, архитектура и приёмка) ·
Инженер (Claude Code, реализация). Спор решает Оператор.

§2. Единицы работы. Задачи с КОДОМ — на ветке task/<id>: зелёные тесты + прогон code-reviewer +
⛔ Куратора → слияние в main. Доки, chore, governance — прямо в main (гейт: вики-линт зелёный + ⛔).
GitHub PR — по желанию для просмотра диффа, не обязателен (приватный репо, Оператор + Куратор).

§3. Формат задачи Инженеру: Фаза · Компонент · Что сделать · Границы (что НЕ трогать) ·
Критерии приёмки · Зависимости.

§4. Решения: только через ADR. ADR короткий: контекст → решение → последствия.

§5. Безопасность: см. wiki/concepts/. Любое изменение, касающееся ключей/доступов,
проходит отдельное ревью Куратора.

§6. Хэндофф: каждая сессия Инженера заканчивается handoff-заметкой в wiki/handoffs/.

§7. Проверка дизайна.
§7.7. Critical Review Cycle: перед фиксацией дизайн-документа или расчётной формулы —
цикл «сомнение → разведка → решение»: пересчитать каждое опорное число руками, прогнать
краевые значения через формулы, прогнать формулу через тест, который потребовал её автор.
«Документ выглядит связно» — не валидация. Минимум один полный проход без расхождений.
