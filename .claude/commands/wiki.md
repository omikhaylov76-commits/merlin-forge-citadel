# /wiki — запомни / найди (библиотекарь)

Два режима. Определи по запросу; неясно — спроси одной строкой.

## Режим ЗАПОМНИ (человек дал ссылку, файл, факт или решение)
1. Пойми природу: внешний сервис/API/библиотека → `wiki/entities/<имя>.md`;
   статья/дока/репозиторий → `wiki/summaries/<имя>.md`; принятое решение →
   `wiki/decisions/NNNN-<слаг>.md`.
2. Если страница про это уже есть — ОБНОВИ её (правило одного места), не плоди дубль.
3. Пиши сжато: только факты, которые понадобятся проекту; ≤150 строк; frontmatter
   `type/title/tags/updated/sources` (sources — откуда взято, обязательно).
4. Добавь однострочник в `wiki/index.md` (≤100 символов) и строку в `wiki/log.md`.
5. Покажи карточку. ⛔ «ок» — если правил существующую страницу с чужими фактами.

## Режим НАЙДИ (человек задал вопрос)
1. Сначала `wiki/index.md` (+теги), открой только релевантные страницы — не всё подряд.
2. Ответь из вики со ссылками на страницы. В вики нет → скажи честно; во внешний поиск иди
   только после «ок» человека.
3. Если ответ потребовал реального синтеза (сводил 3+ страниц или внешний поиск) — предложи:
   «сохранить ответ страницей в concepts/summaries?» Ценные ответы накапливаются, а не испаряются.

## Мини-линт (по запросу «/wiki линт»)
Быстрый проход: битые ссылки в index, дубли фактов, страницы >150 строк, кандидаты в attic
(старше 2 фаз, нет входящих ссылок). Список находок, чинить только после «ок».
