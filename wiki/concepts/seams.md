# Швы платформы (единый источник)

Шов = граница двух миров: что пересекает, в какую сторону, каким механизмом, что при отказе.
Закон №3: прямых импортов между модулями НЕТ; допустимы HTTP API ядра и таблицы-контракты.
Трассировки сценариев по швам — [flows](flows.md). Развилки решены в ADR 0007–0011 (accepted 2026-07-10).

## Карта швов

| # | Шов | Направление | Механизм | Что пересекает |
|---|-----|-------------|----------|----------------|
| S1 | console → core | только → | REST JSON, JWT Оператора (+TOTP) | read-models всего домена; команды CRUD/deploy/pause/stop |
| S2 | portal → core | только → | REST JSON, JWT клиента, узкий роутер `/v1/portal/*` | read-model «моё»: equity, статус, сделки, биллинг; 2 команды (ADR-0005) |
| S3 | orchestrator ↔ core | poll → / callback → | REST long-poll `/v1/internal/jobs/next` + ack, service-token (ADR-0009) | jobs (deploy/teardown), шифроконверты ключей, статусы инстансов |
| S4 | bots ↔ core | push → / poll → | REST push телеметрии + long-poll команд, opaque-токен инстанса | heartbeat/equity/trades/events →; pause/resume/stop_close ← |
| S5 | orchestrator → Railway | только → | GraphQL API (InfraDriver) | serviceCreate(image, env), redeploy, delete; infra_ref обратно |
| S6 | core → БД платформы | только core | SQLAlchemy + Alembic; core — единственный владелец схемы | весь домен; никто другой прав в эту БД не имеет |
| S7 | bot → БД ботов | только свой инстанс | отдельный кластер, схема-на-инстанс, роль со scope схемы (ADR-0007) | внутреннее состояние движка; платформа внутрь не смотрит |
| S8 | клиент → core (key-intake) | одноразово → | форма по одноразовой ссылке, TLS; core шифрует асимметричным конвертом (ADR-0010) | API-ключ биржи (plaintext только в памяти запроса) |

Инвариант направлений: core ни к кому не ходит сам (кроме своей БД и outbox-диспетчера);
боты и оркестратор — клиенты core; фронты — клиенты core. Все стрелки сходятся в ядро.

## Контракты на швах (суть; детали — по ссылкам)

- **S1 console:** полные read-models + команды. Каждая команда = строка audit_log (закон №4).
  Деплой из консоли НЕ зовёт Railway: только `POST /v1/instances` → ядро создаёт job.
- **S2 portal:** отдельные response-модели (pydantic), НЕ переиспользуются операторские.
  Белый список полей: status(упрощённый), health, equity-серия, сделки(символ/сторона/pnl),
  биллинг-сводка. Запрещены навсегда: node_id/infra_ref, profile config, ключи, чужие клиенты,
  операторские заметки. Гарантия — leak-тест в CI: ответ портала не содержит запретных ключей.
  Команды: pause мгновенно; stop_close — серверный challenge (двойное подтверждение не в UI,
  а в API: `POST …/stop_close` → challenge_id → повторный POST ≤60с). Обе — аудит + алерт.
- **S3 jobs:** ядро пишет jobs, оркестратор арендует (lease + lease_expires_at + attempts).
  Идемпотентность: ≤1 активный deploy-job на инстанс (unique partial index); перед serviceCreate
  оркестратор ищет сервис по имени `mfc-inst-{id}` — «усынови или упади», дублей не создаёт.
  Финал job: ack(done|failed, detail). 3 неудачи → failed + алерт, без бесконечных ретраев.
  ✅ **Реализовано MFC-004:** миграция 0003 `jobs`; internal API `GET /internal/jobs/next` (long-poll
  SKIP LOCKED, БЕЗ удержания коннекта SCL1) + `ack` (fencing по `lease_nonce` OPS2; release без штрафа
  OPS16; deploy→teardown-компенсация OPS3; teardown вечно requeue OPS5); продюсеры `POST /v1/instances`+`/teardown`.
- **S4 Контракт Бота v0+:** [bot-contract](bot-contract.md). Уточнения MFC-000:
  ts бота проверяется (|ts−now| < 48ч), авторитетно server-side received_at; дедуп по (instance, ts, канал);
  батчи ограничены (413 → бот дробит); буфер бота при недоступности core — кольцевой, ~24ч,
  trades не выбрасываются первыми; бот мигрирует СВОЮ схему в БД ботов сам при старте;
  contract_version декларируется в heartbeat, v1 только записываем (enforcement Ф2).
  ✅ **Реализовано MFC-005 (core-сторона):** схемы v0 `contracts/*.schema.json`; миграция 0004
  (equity/trades/events/commands); приём телеметрии (`routes_telemetry`, dedup/ts-skew/413, heartbeat→
  last_heartbeat_at) + команды (`routes_commands`, long-poll липкий stop_close + ack, ADR-0005) токеном
  инстанса. ✅ **Картридж paper-bot (клиент этого API) — MFC-006:** эталонная реализация Контракта v0
  (bots/paper-bot) — детерминированный движок (синус+seeded-сделки), честные семантики ADR-0005
  (pause=стоп входов/позиции держатся, stop_close=закрыть+встать); сквозняк вживую пройден. Полный
  кольцевой буфер бота при недоступности core (~24ч) — отложен (v1 best-effort).
- **S5 InfraDriver:** интерфейс `deploy(spec) → infra_ref` · `destroy(infra_ref)` · `status(infra_ref)`.
  infra_ref = «railway:{project}:{svc}» — проект в формате СРАЗУ (SCL8: квота сервисов/проект вынудит
  мульти-проект, миграция формата потом дорога). restartPolicy=never для бот-сервисов (OPS1). destroy
  идемпотентен: нет сервиса = успех (OPS5). RailwayDriver v1; DockerDriver — заглушка с conformance-тестом.
  ✅ **Реализовано MFC-004:** ABC `InfraDriver` + `FakeDriver` (in-memory, тесты/сквозняк) + `DockerDriver`
  (шумная заглушка) + `RailwayDriver` (GraphQL усынови-или-создай; ⚠️ точная схема — на боевой обкатке,
  отдельная веха). Квоты Railway (сервисов/проект, rate-limit) — измерить на обкатке (ADR-0003).
- **S6/S7 БД:** две РАЗНЫЕ базы. Платформенная — только core (Alembic, закон №7). Ботовая —
  отдельный кластер: оркестратор провижионит схему+роль при деплое, движок владеет содержимым
  своей схемы. Физически подтверждает угрозу №4: у бота нет пути к данным платформы.

## Отказы на швах (что делает сосед, когда шов молчит)

| Отказ | Обнаружение | Реакция v1 |
|---|---|---|
| Бот молчит (heartbeat) | нет 3×60с | health=stale, алерт Оператору; АВТОРЕСТАРТА НЕТ (позиции неизвестны) — runbook |
| Бот не стартовал | нет 1-го heartbeat за 5 мин | job=failed, instance=failed_deploy, алерт + ссылка на логи Railway |
| Job завис | lease истёк | возвращается в очередь, attempts++; 3 → failed + алерт |
| Расшифровка ключа не удалась | исключение в оркестраторе | job=failed БЕЗ ретрая, аудит, алерт; ключ — на повторный intake |
| Двойной деплой | unique index + поиск по имени | второй job отвергнут ядром; сирота-сервис усыновляется по имени |
| Railway API лёг | ошибки GraphQL | backoff+jitter, lease отпущен; >15 мин → алерт. Торговля ботов НЕ страдает |
| core лёг | внешний uptime-пинг /healthz | боты копят телеметрию и торгуют по своим стопам; kill-switch клиента деградирует (flows Б.5) |
| Railway регион/compute лёг | нет телеметрии массово | АГРЕГИРОВАННЫЙ алерт (не по инстансу); флот держат только вшитые стопы; «СТОП ФЛОТ» — runbook массового отказа |
| Оркестратор лёг (нет входа) | метрика «возраст старейшего pending job» + heartbeat оркестратора | алерт; jobs не арендуются — deploy/teardown стоят (OPS7) |
| stop_close не ack'нут | таймаут cmd 10 мин | алерт + runbook «закрыть руками»; портал честно показывает стадию |
| Сделка при status≠running | инвариант приёмника | trade при paused/stopping → алерт + audit (OPS14) |
| Ключ протух у живого инстанса | event key_perms / поток auth-error | алерт + автопауза инстанса (OPS15) |
| Флуд/дубли телеметрии | rate-limit на токен, дедуп | 429/дедуп тихо; повторяющийся флуд — алерт |

Инвариант каналов (SCL1): long-poll (S3/S4) НЕ держит БД-сессию/транзакцию — иначе флот вешает пул ядра.
Reconcile-цикл (OPS13): периодически сверять instances ↔ сервисы `mfc-inst-*`; сироты → алерт+компенсация.
Алерты (SCL3): outbox с severity, приоритетная выборка (команды клиента/доступ к ключам — вне очереди),
коалесценция однотипных, dead-man (тик диспетчера в /healthz — реализован в MFC-002: блок `scheduler`, ADR-0012). stale-скан heartbeat реализован (MFC-003: свёртка часового ставит health ok/stale/dead + audit; доставка алерта — с outbox). Полный триаж отказов — seams-review.md.

## Трубы-заглушки (что лежит в фундаменте закрытым)

| Труба | Реально в v1 | Заглушка и как открывается |
|---|---|---|
| Биржи | bybit | enum + таблица exchange_capabilities (1 строка); OKX/BitGet = +строка +поддержка в образе бота |
| Ансамбли | ensemble_id nullable + таблица | скоуп 'ensemble' в дизайне токенов; v1 API отвергает не-instance скоупы явной ошибкой + тест-гвоздь |
| DockerDriver | интерфейс | файл NotImplemented + conformance-тест; включение = реализация методов |
| Бэктест-jobs | kind зарезервирован | v1 ядро отвергает kind=backtest; включение Ф5 |
| Outbox | таблица events + 1 диспетчер-цикл | v1 единственный потребитель — Telegram-алерты; не Kafka и не станет |
| Кокпит Пифагора | — | авторизованный прокси консоли (ADR-0002), появится в Ф2 |

Правило: заглушка = явная ошибка + тест, который её гвоздит. Молчаливых полудорог нет.
