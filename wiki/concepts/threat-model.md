# Модель угроз (рабочий список)
1. Утечка ключей биржи → конверт-шифрование (асимметричная master-пара ADR-0010: приватная
   половина только у оркестратора), права ключей «только торговля», вывод запрещён на бирже,
   IP-whitelist (на Railway — общий static IP, риск принят ADR-0003; whitelist — после включения
   Pro, до того риск принят, см. keys-policy). ⚠️ Живой plaintext-ключ лежит в переменных сервиса
   Railway весь срок инстанса (сток, SEC2) — компенсация: права ключа + очистка vars на teardown/ротации.
2. Захват консоли → 2FA Оператора, короткие сессии, rate-limit, audit_log, алерт на вход с нового устройства.
3. Портал как дыра → у портала нет прав: read-only витрина + 2 команды через core с аудитом.
4. Бот скомпрометирован → у токена инстанса скоуп только на себя (проверка владения на ВСЕХ
   ручках, не только портал — SEC7); ботам не выдаётся доступ к БД ПЛАТФОРМЫ; в кластере ботов —
   роль только на свою схему (ADR-0007), её кред — секрет инстанса; входящих портов нет.
   Телеметрия бота — недоверенный ввод (экранировать в консоли/портале, SEC5).
5. Внутренняя ошибка (наша) двоит ордера → идемпотентность команд, singleton-инварианты в движке (у Пифагора есть),
   стоп-вшит-в-ордер как закон для картриджей.
6. Платформа-хостинг (Railway) → доверие поставщику принято осознанно; секреты дополнительно шифруются на нашем уровне;
   план Б — DockerDriver/VPS.
7. Клиентские претензии → паспорт с честными числами ДО старта, договор/дисклеймеры (60_legal), полный аудит действий.
8. Оркестратор — главный секретоноситель (приватная половина конверта ADR-0010 + admin БД ботов
   + токен Railway) → компенсация: НЕТ входящих портов (poll-only), секреты только в env,
   минимум зависимостей. IR при компрометации (SEC9): перепаковка DEK НЕ отзывает уже утёкший
   plaintext → немедленный отзыв/перевыпуск ВСЕХ ключей бирж клиентами на биржах + ротация всех
   машинных токенов + ротация master-пары.
9. Скомпрометированный core направляет расшифровку в свой образ (SEC1) → распечатка DEK привязана
   к allowlist-digest образа (ADR-0010v2); регистрация bot_type и назначение образа — аудит.
