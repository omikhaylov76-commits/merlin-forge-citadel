# Модель угроз (рабочий список)
1. Утечка ключей биржи → конверт-шифрование, master-key только у оркестратора, права ключей
   «только торговля», вывод запрещён на бирже, IP-whitelist (на Railway — общий static IP, риск принят ADR-0003).
2. Захват консоли → 2FA Оператора, короткие сессии, rate-limit, audit_log, алерт на вход с нового устройства.
3. Портал как дыра → у портала нет прав: read-only витрина + 2 команды через core с аудитом.
4. Бот скомпрометирован → у токена инстанса скоуп только на себя; ботам не выдаётся доступ к БД;
   входящих портов нет.
5. Внутренняя ошибка (наша) двоит ордера → идемпотентность команд, singleton-инварианты в движке (у Пифагора есть),
   стоп-вшит-в-ордер как закон для картриджей.
6. Платформа-хостинг (Railway) → доверие поставщику принято осознанно; секреты дополнительно шифруются на нашем уровне;
   план Б — DockerDriver/VPS.
7. Клиентские претензии → паспорт с честными числами ДО старта, договор/дисклеймеры (60_legal), полный аудит действий.
