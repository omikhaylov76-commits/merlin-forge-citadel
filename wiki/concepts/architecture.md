# Архитектура — обзор

Четыре модуля вокруг одной БД, боты — за Контрактом:

    ┌─────────┐   ┌────────────┐   ┌──────────┐
    │ console │──▶│            │◀──│  portal  │      боты (контейнеры)
    └─────────┘   │  core API  │   └──────────┘      ┌────┐ ┌────┐ ┌────┐
                  │  Postgres  │◀────телеметрия──────│ #1 │ │ #2 │ │ …  │
    ┌───────────┐ │  audit     │─────команды────────▶└────┘ └────┘ └────┘
    │orchestr.  │◀▶            │
    └─────┬─────┘ └────────────┘
          └─ драйвер: Railway API (v1) / Docker (позже)

Потоки: консоль и портал ходят ТОЛЬКО в core API. Оркестратор арендует jobs у ядра
(internal API, long-poll — ADR-0009) и управляет контейнерами через InfraDriver.
Боты шлют телеметрию в core (push, токен инстанса) и получают команды оттуда же
(long-poll — без входящих портов у ботов). Все внешние стрелки сходятся в ядро; наружу
core инициирует только два исходящих: outbox-диспетчер (алерты Оператору) и разовый
вызов биржи в окне интейка ключа (стартовая проверка прав). Внутри ядра — **scheduler**
(«часовой», один цикл): stale-скан heartbeat, таймауты старта/ack, истечение lease/challenge,
чистка схем (SCL4). Long-poll НЕ держит БД-коннект (SCL1).

Каталог границ (кто/что/как пересекает, отказы, заглушки): **[seams](seams.md)** —
единый источник по швам. Сквозные трассировки и машина состояний инстанса
(status = намерение, health = свежесть heartbeat): **[flows](flows.md)**.

Домен (ядро): clients · exchange_accounts (ключи шифрованы, асимметричный конверт —
ADR-0010) · bot_types · profiles(+паспорта) · instances (status+health, infra_ref,
ensemble_id — труба) · telemetry (heartbeat, equity, trades, events) · billing_periods (HWM,
закрытие — со сверкой по бирже) · research_artifacts · jobs (lease/ack) · audit_log · users/roles.

Принципалы API (ADR-0008 v2): ВСЕ — opaque-токены (hash в БД, скользящий TTL, мгновенный отзыв);
Оператор дополнительно TOTP; клиент — узкий роутер портала; машины — скоуп instance / ensemble-труба
(v1 отвергается) / orchestrator. Проверка владения — на всех ручках.

Данные: ДВЕ БД. Платформенная — единственный владелец core (Alembic). Кластер ботов —
схема-на-инстанс, провижионит оркестратор, движок владеет своей схемой (ADR-0007).

Трубы, лежащие в фундаменте с v1 (как заглушено — таблица в seams): драйвер оркестрации;
поле exchange + таблица возможностей бирж; ensemble_id + скоуп группы (дирижёр = просто
тип бота); outbox-события (v1 — только Telegram-алерты); infra_ref; RBAC; jobs-очередь
(бэктесты позже). Правило заглушки: явная ошибка + тест-гвоздь, молчаливых полудорог нет.
