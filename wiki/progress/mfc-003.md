# MFC-003 — instances + первая боевая свёртка часового (stale-скан)

**Веха:** Ф1 · **Ветка:** task/mfc-003 · **База:** 40dc7bb

## Цель
Материализовать таблицу `instances` (центральная таблица домена) и дать часовому (MFC-002) его
**первую реальную свёртку** — stale-скан heartbeat, отложенную в MFC-002. Свёртка периодически
сканирует инстансы и по свежести `last_heartbeat_at` выставляет `health` ok→stale→dead (flows:64,
паттерн 3×60с из seams:62). Переход health — строка `audit_log` (закон №4). Свёртка = sync-функция,
часовой гоняет её через `to_thread` (не блокируя цикл) — шов, заложенный в MFC-002.

## Развилка на ⛔ — 4 FK инстанса ведут в ещё-не-существующие таблицы
`instances` ссылается на clients / exchange_accounts / bot_types / profiles (все NOT NULL по дизайну),
а их таблиц нет. Варианты:
- **(A, рекомендую)** instances материализуется автономно: столбцы-ссылки остаются (NOT NULL,
  UUID), но **FK-constraint'ы отложены** — включатся, когда родители появятся со своими фичами
  (clients → Ф3 CRM, exchange_accounts → Ф2 ключи, bot_types/profiles → Ф5 Кузница). Осознанно,
  документируется **ADR-0013**. Совпадает с уже принятым YAGNI-курсом (0001 — «НЕ весь домен»).
  Тесты вставляют инстансы со случайными UUID-родителями. Компактно и доказуемо.
- **(B)** материализовать всю FK-замкнутость сразу (clients, bot_types, profiles, exchange_accounts,
  instances = 5 таблиц). Честные FK, но 4 таблицы без потребителя в v1 (спекулятивная схема —
  «нечем доказать», урок MFC-002). Объёмнее.

Что делаем СЕЙЧАС включаем в любом варианте: партиал-уникальный индекс «≤1 живой инстанс на
account_id» (OPS3/MON2, это и хвост MFC-001-доп); CHECK на health и status.

## Границы (чего в MFC-003 НЕТ)
- НЕТ таблиц clients/jobs/telemetry/outbox — только instances (+ ссылки-заглушки по варианту A).
- НЕТ Telegram/outbox-алерта: stale → переход health + строка audit (actor=`system:sentinel`);
  доставка алерта — с фичей outbox_events (позже). seams:62 «алерт Оператору» закрыт лишь частично.
- НЕТ авто-рестарта/действий над инстансом (позиции неизвестны, seams:62).
- НЕТ ручек API по инстансам (это оркестратор/консоль позже).
