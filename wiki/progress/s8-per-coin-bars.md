# Проводка per-coin баров — ПЛАН (подпись Куратора есть, QUEUE 2026-07-22)

Куратор ПОДПИСАЛ (сверил метод в коде `scout/bars.py` — volnorm = статистика характера монеты, НЕ
переобучение) + дал **3 уточнения ДО кода** + **1 проверку** (VVV — закрыта). Only engine-режим Борса.

## Что открыла разведка (ключевое)

Вендор **УЖЕ считает и хранит per-coin бары** — их не надо изобретать, только перестать выбрасывать:
- `scout/bars.py::volnorm_bars(p75,p90)`: `mb1=clamp(snap(K1·P75),1.5,3.5)`, `mb2=clamp(snap(K2·P90),
  max(2.5,mb1+0.5),6.0)`; K1=0.78/K2=0.95 — **глобальные** константы (НЕ per-coin) × квантиль волы
  монеты по рабочему окну. Docstring авторов: «честный метод (капкан R3 — НЕ подбор баров per-coin)».
- `scout/bars.py::bars_for`: config-override (16 боевых монет) → `bar_source="config"`; иначе volnorm
  → `"volnorm-v1"`. Оговорка Куратора: config-16 исторически из перф-свипа (помечены «не held-out»),
  но проводка это НЕ ухудшает — они так и торговали; остальные 139 получат ЧЕСТНЫЙ volnorm вместо плоских 2.0/3.5.
- `storage/db.py`: таблица `scout_list(symbol,score,mb1,mb2,bar_source,payload)` — `scout_list_all()`
  их отдаёт. Наш `scan_list_rows()` берёт только `{symbol,score}` → **mb1/mb2/bar_source выбрасываются**.
- **Линчпин подтверждён против вендора** (`config/strategy.py::_load_coins_config`+`_coerce_coins`):
  движок читает coins.json и берёт per-coin `mb1/mb2` ЦЕЛИКОМ (REPLACE вселенной). Валидация: mb —
  числа, symbol …USDT, диапазоны (`config.validate`: mb1/mb2>0). volnorm-клампы внутри → валидно.

⇒ Проводка = «перестать выбрасывать mb + писать per-coin в coins.json». Форс `_DEFAULT_COIN` в
`placeable_scan` заменяется на per-coin mb (адаптерный вердикт == реальная постановка движка сохраняется).

## Слои постройки

### Слой 1 — scout_reader (пул несёт бары)
- `scan_list_rows()`: добавить `mb1/mb2/bar_source` в строки (читаются из `scout_list_all`, сейчас дропаются).
- `placeable_scan()`: вместо форса `_DEFAULT_COIN` — ставить в `COINS_CONFIG[sym]` per-coin mb строки
  (fallback `_DEFAULT_COIN` mb, если строка без mb), classify считает ЕЮ; вернуть mb в дескрипторе
  placeable → провайдер донесёт в стек. Убирает адверс-хак (вердикт == постановка, т.к. пишем ту же mb).

### Слой 2 — dynamic_universe (стек хранит + пишет бары)
- `_recompute_engine(placeable, now_mono)`: `fresh[sym]` несёт mb; **sticky-логика** (уточнение Куратора):
  - held-монета — бары ЗАМОРОЖЕНЫ (те, с которыми вошла), пока позиция жива (уточнение #2);
  - не-held стек-монета — mb sticky (не дёргаем каждый скан), пере-захват только по редкому ритму refresh;
  - новый вход — захват текущей mb пула.
- `_maybe_write`: строит per-coin блоки (fallback `_DEFAULT_COIN`, если mb нет) + **сигнатура по
  СОДЕРЖИМОМУ** (symbol+mb), не только по символам (уточнение #1: иначе смена mb при том же составе
  не запишется — тихая гниль). min-интервал/positions-flag/пол-на-пустоту не трогаем.
- `view()`: отдать mb1/mb2/bar_source в items стека → карточка Борса покажет бары per-coin (визуальная сверка).

### Слой 3 — конфиг/частота
- `config.py`: `dynamic_bars_refresh_s` (env `DYNAMIC_BARS_REFRESH_S`, дефолт ~30д=2592000) — decouple
  ритма обновления баров от Этапа A (Оператор: реже, «раз в месяц — стабильнее пороги»; уточнение из QUEUE).

## 3 уточнения Куратора (впаяны в план ↑)
1. **`_maybe_write` по mb-контенту**, не только символам (тихая гниль) — но без дёрганья каждый чих
   (mb меняется по редкому ритму sticky+refresh; предохранители min-write/positions-flag на месте).
2. **Held-монета: бары sticky** (те, с которыми вошла), фолбэк `_DEFAULT_COIN` — только если mb вообще
   неоткуда взять. Месячный пересчёт held не трогает.
3. **Окно наблюдения перед деплоем.** Engine-источник принят по ОДНОМУ снимку — тонко. Бары строим
   СЕЙЧАС, деплой Борса — после спокойного окна (несколько 4h-границ engine-источника без осцилляций;
   не смешивать два изменения поведения в один день).
