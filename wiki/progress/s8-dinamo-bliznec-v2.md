# S8 «Динамо-близнец» · Веха 2 (демо-ордера из динамики)

**Статус: ПЛАН, ждёт подписи Куратора + гейт Оператор+Куратор на живую торговлю.** Веха 1 закрыта вживую
(Борс `cd8d0534` в динамике, dry-run, стек из печки, 0 ордеров — `[DRY-RUN] поставил БЫ`). Веха 2 = снять
dry-run у Борса, чтобы он реально ставил демо-ордера по динамическим монетам — **безопасно**.

## Цель (критерий приёмки)
Борс ставит демо-ордера (`LIVE_TRADING_ENABLED=1`, `BYBIT_DEMO=1`) по монетам ДИНАМИЧЕСКОГО набора, и
**ни одна монета с открытой позицией/ордером не выпадает из набора и не теряется при смене вселенной**.
Персиваль/Галахад/эталон не задеты. Реальные ключи/деньги — НЕ здесь (отдельный go-live).

## Почему нужен ПИН (первопричина, ADR-0019 гейт)
Сейчас триппвайр `_make_dynamic_provider` **запирает** `DYNAMIC+LIVE` — by design. Причина: сетап дошёл
до входа → у обходчика движка `state='committed'` (в находки НЕ эмитится) → исчезает из печки →
`missed`→`exit_scans` → провайдер выкинул символ → супервизор рестартнул движок БЕЗ него → **позиция
брошена**. Гейт Вехи 2 (ADR-0019) двухчастный, ОБЕ части обязательны:
- **(а) не рестартить движок при открытых позициях/ордерах**;
- **(б) пришпилить символ с живой позицией/ордером в наборе** (pin поверх `exit_scans`) до полного финала.

## Под-шаги

### 1. Провайдер узнаёт про позиции/ордера
- [ ] 1.1 `bot.py` уже строит engine_state (позиции/ордера через `build_monitor`/`orders_open_all`) каждый
      тик → передавать провайдеру **множество символов с живой позицией/ордером** в `provider.tick(now, held)`.
      Провайдер не лезет в БД сам (шов чист: адаптер читает, провайдер получает готовое множество).
- [ ] 1.2 Нормализация символов held (upper) == нормализация стека.

### 2. ПИН (часть «б»)
- [ ] 2.1 `dynamic_universe._recompute`: символы из `held` **не выкидываются** по `exit_scans`/исчезновению
      из печки — они ПРИШПИЛЕНЫ, остаются в `_stack` пока в `held`. Слот занят (определение Оператора: пока
      по символу есть живая позиция/ордер).
- [ ] 2.2 `_maybe_write`: записанный набор ВСЕГДА включает `held` (даже если выпали из печки) → coins.json
      никогда не роняет монету с позицией → движок после рестарта её видит и перечитывает позицию из БД.
- [ ] 2.3 `view()`: пометить пришпиленные (напр. `pinned: true`) для карточки (Оператор видит «держим под позицию»).

### 3. Гейт рестарта (часть «а»)
- [ ] 3.1 `start.sh engine_supervise`: перед мягким рестартом на смену `.gen` — **проверить открытые
      позиции**. Механизм-развилка (F-restart ниже): либо адаптер пишет флаг-файл `positions_open`,
      супервизор его читает; либо провайдер НЕ бампает `.gen`, пока смена уронила бы held (с пином —
      held всегда в наборе, так что смена безопасна, но рестарт мид-трейд лучше отложить).
- [ ] 3.2 Заменить no-op-заглушку (`start.sh:47` TODO) на рабочий гейт: позиции открыты → отложить рестарт
      (до закрытия ИЛИ max-wait с логом), не бросать движок мид-ордер.

### 4. Безопасное снятие триппвайра
- [ ] 4.1 Когда (а)+(б) реализованы и покрыты тестами — снять триппвайр (`_make_dynamic_provider` разрешает
      `DYNAMIC+LIVE`). Оставить осознанный гейт (напр. `PIN_READY=1` или просто удаление после подписи).
- [ ] 4.2 ADR-0019: обновить гейт Вехи 2 с «двухчастный, обязателен» на «реализован» + реестр.

### 5. Тесты (против настоящего вендора — урок S7)
- [ ] 5.1 Пин: символ в `held` НЕ выкидывается по exit_scans; остаётся в наборе; coins.json его несёт.
- [ ] 5.2 Пин снят: символ ушёл из `held` (позиция закрылась) → нормальный exit по гистерезису.
- [ ] 5.3 Гейт рестарта: позиции открыты → рестарт отложен; закрылись → рестарт прошёл.
- [ ] 5.4 held читается из НАСТОЯЩЕГО build_monitor/orders (не мок).

### 6. Деплой + живая сверка (гейт Оператор+Куратор)
- [ ] 6.1 ⛔ **ГЕЙТ: явная подпись Оператора И Куратора на `LIVE_TRADING_ENABLED=1`** (демо-ордера).
- [ ] 6.2 `rw_deploy_bors.py` (или ветка): `LIVE_TRADING_ENABLED=1` на Борса (+ починить его ключ — план А
      хвост: вписать свой BORS-ключ) → редеплой только Борса. Стражи Персиваля/Галахада внутри.
- [ ] 6.3 Живая сверка: Борс ставит демо-ордер по динамической монете (`[DRY-RUN] поставил БЫ` → реальный
      демо-ордер); монета с позицией пришпилена (не выпадает при смене вселенной); движок не рестартнул
      мид-позицию; Персиваль/Галахад не задеты. Скрин карточки Борса с позицией+пином.

## Развилки Куратору (подпись до кода)
- **F-restart (главная):** механизм гейта рестарта в shell-супервизоре. (A) адаптер пишет флаг-файл
  `positions_open` (супервизор `[ -s ]` перед рестартом); (B) с пином held всегда в наборе → рестарт
  безопасен, гейт лишь откладывает мид-трейд. Реком: **A + пин** (двойная защита, дух ADR-0019 в.4).
- **F-pin-scope:** «полный финал» слота = определение Оператора (позиция закрыта И ордера отменены). Как
  адаптер это детектит (build_monitor position=null И orders_open пусты по символу).
- **F-key:** починить общий ключ Борса на его BORS-ключ в этом же деплое (план А хвост) — или отдельно.
- **F-tripwire:** снять триппвайр совсем или оставить `PIN_READY`-гейт (реком: снять после подписи, гейт=подпись).
