# S8 «Динамо-близнец» · Веха 2 (демо-ордера из динамики)

**Статус: ✍️ РАЗВИЛКИ ПОДПИСАНЫ Куратором (2026-07-18) → КОД В РАБОТЕ на ветке `task/s8-veha2-pin`.**
Гейт `LIVE_TRADING=1` (Оператор+Куратор) — ПОСЛЕ merge. Веха 1 закрыта вживую (Борс `cd8d0534` в динамике,
dry-run, стек из печки, 0 ордеров — `[DRY-RUN] поставил БЫ`). Веха 2 = снять dry-run у Борса, чтобы он реально
ставил демо-ордера по динамическим монетам — **безопасно** (пин + гейт рестарта, двойная защита).

**Рамка Оператора:** запуск «по-чистому» = СТРЕСС-ТЕСТ живучести движка, проверяем что ВСЕ предохранители
на месте и держат (не гонка за ордером). Каждый предохранитель — под наблюдением на живой сверке.

## Цель (критерий приёмки)
Борс ставит демо-ордера (`LIVE_TRADING_ENABLED=1`, `BYBIT_DEMO=1`) по монетам ДИНАМИЧЕСКОГО набора, и
**ни одна монета с открытой позицией/ордером не выпадает из набора и не теряется при смене вселенной**.
Персиваль/Галахад/эталон не задеты. Реальные ключи/деньги — НЕ здесь (отдельный go-live).

## Почему нужен ПИН (первопричина, ADR-0019 гейт)
Сейчас триппвайр `_make_dynamic_provider` **запирает** `DYNAMIC+LIVE` — by design. Причина: сетап дошёл
до входа → у обходчика движка `state='committed'` (в находки НЕ эмитится) → исчезает из печки →
`missed`→`exit_scans` → провайдер выкинул символ → супервизор рестартнул движок БЕЗ него → **позиция
брошена**. Гейт Вехи 2 (ADR-0019) двухчастный, ОБЕ части обязательны:
- **(а) не рестартить движок при открытых позициях/ордерах**;
- **(б) пришпилить символ с живой позицией/ордером в наборе** (pin поверх `exit_scans`) до полного финала.

## Развилки — ✍️ ПОДПИСАНЫ Куратором (2026-07-18, QUEUE)
- **F-restart** ✅ **A + пин (двойная защита).** Адаптер пишет флаг-файл открытых позиций/ордеров →
  супервизор `[ -s ]` ПЕРЕД мягким рестартом → рестарт при живой позиции **откладывается** (до закрытия
  ИЛИ max-wait с громким логом), движок не бросается мид-ордер. Плюс пин держит held всегда в наборе.
  Не «либо-либо», оба слоя — money-контур (пусть демо) стоит двойной страховки. Дух ADR-0019 в.4.
- **F-pin-scope** ✅ (определение Оператора, дословно): слот занят, монета ПРИШПИЛЕНА, пока есть
  **открытая позиция ИЛИ висящий ордер**. Свободен ТОЛЬКО когда позиция закрыта **И** все ордера сняты
  (полный финал). Частичный (первая нога закрылась, глубокие ждут) = слот ЗАНЯТ. Детект: `build_monitor`
  position по символу пусто **И** orders_open по символу пусто → отпуск. **Читать ОБА источника.**
- **F-key** ✅ **отдельным шагом позже, НЕ в этом деплое.** Общий demo-ключ (Борс на GAWAIN-слоте) для
  демо-ордеров Вехи 2 приемлем (демо, учебный счёт). Разводку ключей флота — отдельной задачей ПЕРЕД
  реальным go-live. Борс идёт на текущем рабочем ключе, как в Вехе 1. Одна переменная за раз.
- **F-tripwire** ✅ **снять после (а)+(б)+тестов, гейт = подпись.** Без `PIN_READY`-костыля: триппвайр
  удаляется, его роль берёт явный гейт «подпись Оператора+Куратора на LIVE». ADR-0019 гейт → «реализован».

## ➕ Наблюдение-тест (по слову Оператора — НЕ код, пункт приёмки на живой сверке)
Движок УЖЕ отказывается повторно входить в отработавший сетап (`warm.py`: авто берёт только
`auto_eligible` = нетронутый PENDING, все 3 ноги лимитами; OPEN/частичный → не авто). На живой сверке
ДОКАЗАТЬ наблюдением: сетап, где первая нога отработала и тейк снят, при смене вселенной/рестарте НЕ
берётся заново (движок видит позицию из БД, класс ≠ нетронутый). Дыра в наблюдении → стоп, доклад, чиним.
