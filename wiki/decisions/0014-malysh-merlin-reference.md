# ADR-0014. «Малыш Мерлин» — эталон-нулевой-пациент — accepted 2026-07-12

Контекст: Пифагор — первый боевой движок за Контрактом Бота. Картридж бежит из тонкого
vendored-субсета (живой граф, без `engine/`-эталона — ADR-0001, разбор #13). Нужна платформенная
гарантия, что ОРИГИНАЛ движка на снимке v8.3 `b75bd17` (полный `b75bd176193abba1803899c092052c1f9c00a9eb`)
не потеряется и не «поплывёт»: ни код, ни заводские настройки. Оригинал `pifagor-v81` по закону №6/
границам Куратора не редактируется — только снимок и чтение. Имя дал Оператор: «Малыш Мерлин».

Решение: завести ДВЕ независимые страховки оригинала + залоченный профиль + версионирование.

**A. Полный замороженный архив (дизастер-рекавери, «железный дубликат») в независимых местах:**
1. Неснимаемый **annotated git-тег `malysh-merlin/v8.3-b75bd17`** на репо Пифагора — вечный
   указатель на точный коммит + GitHub release там же. Код Пифагора НЕ меняется (тег — не правка).
2. Физический архив ВНЕ репо Пифагора: `git archive b75bd17` → `malysh-merlin-v8.3-b75bd17.tar.gz`
   (весь код, включая `engine/`), хранимый **release-ассетом Цитадели** (не в git-дереве — раздул бы
   историю; git-lfs не подключён, решение F1 #21). **SHA256 архива** зафиксирован в `reference/README.md`
   (верификация целостности; SHA загруженного ассета сверяется с записанным).
Третья фактическая копия — локальный клон Оператора вне GitHub (#21). Это НЕ рантайм-код (картридж
бежит из субсета) — только страховка.

**B. Залоченный профиль-эталон (нулевой пациент — настройки):** заводские дефолты Пифагора @b75bd17
(23 крутилки риск/капитал/исполнение/стратегия/ops) заморожены в read-only
`reference/malysh-merlin-profile-v8.3.json`, извлечены детерминированно из `config.knobs` чистого
клона (без запуска движка). Инвариант **«клонируй — не редактируй»**: эталон не правят, с него
клонируют (в UI «Сохранить» на эталоне → принудительно «Сохранить как копию»). Любой эксперимент =
форк, всегда сравним с оригиналом. Имя в платформе: «Малыш Мерлин» (эталон · нулевой пациент · v8.3).

**C. Версионирование снимка движка:** `b75bd17` = «Пифагор v8.3, эталонный снимок». Будущее улучшение
движка = НОВАЯ версия (v8.4…); эталон b75bd17 не перезатирается никогда.

Сейчас vs Ф5: артефакты A+B и захват дефолтов — СЕЙЧАС (не зависят от готовности картриджа, #11).
Полный залок профиля в UI (enforcement «сохранить как копию») — с Ф5 (Кузница-UI). До неё «залок» =
файловый read-only артефакт + этот инвариант.

Отклонено: (а) держать архив в git-дереве без LFS — раздувает историю ~1.5 MB навсегда; (б) хранить
только тег без физического архива — не переживает исчезновение репо Пифагора (нужна самодостаточность
Цитадели); (в) выдуманный/синтетический паспорт профиля — паспорт эталона обязан быть реальным
трек-рекордом (закон Кузницы D8, паспорт без OOS не существует), поэтому фиксируем фактические дефолты,
а не сочинённые.
