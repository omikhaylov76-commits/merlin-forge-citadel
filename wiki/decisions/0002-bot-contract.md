# ADR-0002. Контракт Бота v0
Решение: три канала. (1) Вход: env-переменные (конфиг профиля + секреты + токен инстанса).
(2) Телеметрия: HTTP push в core API (heartbeat ≤60с; equity-точки; сделки; события) по схеме
contracts/telemetry-schema. (3) Команды: бот опрашивает core (long-poll) — start/pause/stop_close.
Никаких входящих портов у ботов; никакого прямого доступа ботов к общей БД.
Последствия: дирижёр-бот возможен без новых механизмов (читает телеметрию группы, шлёт команды
через тот же API со скоупом группы). Дашборд «под капотом» для Пифагора — отдельный порт его
кокпита через авторизованный прокси консоли (только Оператор).

**Расширен [ADR-0016](0016-scout-channel-onboard-scout.md) (Контракт v1):** к телеметрии добавлен
ПЯТЫЙ канал — **scout** (движко-нейтральный replace-снимок сетапов per instance/symbol/tf); онбординг
штатного keyless-скаута из обёртки картриджа (закон №6 уточнён). CONTRACT_VERSION v0→v1.
