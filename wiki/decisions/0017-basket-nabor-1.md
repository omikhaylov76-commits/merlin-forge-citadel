# ADR-0017 — Набор Оператора (НАБОР-1): витрина + хранение

## Контекст
Видение Оператора (QUEUE #1080–1085): рабочий цикл «разведка → отбор → работа». Оператор отмечает
сетап звёздочкой → монета с контекстом складывается в «Набор» → позже (НАБОР-2, боевой мост, ОТДЕЛЬНОЙ
спекой Куратора) набор уезжает боту как рабочий список монет. Форму НАБОР-1 Куратор зафиксировал
(#1082); строится по слову Оператора «поехали» (#S7, канал синхронизируется через QUEUE).

**НАБОР-1 — только безопасная половина: витрина + хранение. НИЧЕГО не торгует.** Передача боту
(изменение рабочего списка Персиваля) — НАБОР-2, вне этого ADR, строго со спекой Куратора и явным «да»
Оператора на каждый запуск.

## Решение
Аддитивно, по швам (ядро ↔ HTTP API ↔ консоль-дисплей; закон №3):

1. **Таблица `basket_items`** (миграция 0012, аддитивна): `id, symbol, tf, source(scout|screener),
   context JSONB, note, added_by, created_at`. **Глобальная корзина Оператора** — НЕ per-instance
   (это отбор Оператора, а не телеметрия бота; позже питает НАБОР-2). `uniq (symbol, tf)` —
   повторная звёздочка **upsert'ит** контекст, не плодит дубли (ключ звёздочки «в наборе?»).
2. **`context` — недоверенный JSON** снимка сетапа (скор/стадия/A-B/входы/детектор), как scout/screener
   (#53): экранируется на ВЫВОДЕ (React-фронт), в ядре не парсится.
3. **`added_by` — свободная ссылка** на актора (UUID без жёсткого FK на users), как `audit_log.actor`:
   integrity-паттерн проекта + не ломает TRUNCATE в тестах.
4. **Роутер `/v1/basket`** (операторский `require_role("operator")`; портал прав НЕ имеет, закон №5):
   GET список · POST добавить(upsert) · DELETE убрать. Каждое add/remove — строка `audit_log` (закон №4).
5. **Консоль**: пункт «Набор» в Кузнице + экран-список (монета/ТФ/источник/скор/стадия/время + «убрать»);
   звёздочка-тумблер на детали сетапа Разведки (ScoutDetail) → add/remove, состояние из GET /basket.

## Альтернативы (отвергнуто)
- **Per-instance корзина** — набор привязан к боту. Отвергнуто: Набор — отбор Оператора поверх флота,
  один; привязка к инстансу усложнит НАБОР-2 (какому боту отдать — решает Оператор при запуске, не хранение).
- **Дедуп по (symbol, tf, source)** — дубли одной монеты из разных источников. Отвергнуто: Оператор мыслит
  монетой, звёздочка «в наборе?» на сетапе должна быть однозначной по (монета, ТФ).
- **Мягкое хранение на фронте (localStorage)** — запрещено в консоли (#32) + не переживает вкладки/устройства
  + НАБОР-2 читает из ядра. Хранение — в ядре.

## Последствия
- +1 аддитивная таблица, 0 изменений существующих; боты/картридж НЕ трогаются (Персиваль вне игры).
- НАБОР-2 (боевой мост) получит готовый источник (basket_items) — читать enabled-набор монет + контекст.
- Звёздочка Скринера (source=screener) — следующим касанием (форма finding иная); MVP покрывает Разведку.
- Каскад при удалении инстанса Набора не касается (глобальный, без FK на instances).
