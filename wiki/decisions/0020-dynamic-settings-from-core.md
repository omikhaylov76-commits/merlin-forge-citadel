# ADR-0020 — Канал критериев Динамики из ядра (S8 «Динамо-близнец», Слой 3)

## Контекст
Разъём вселенной (ADR-0019) даёт боту брать монеты из печки, а провайдер (Слой 2) держит стек. Но
критерии ОТБОРА (какой скор пускать в стек, какая свежесть, сколько монет держать) должны задаваться
Оператором в консоли (раздел 5 «Динамика» Конструктора), а не жить только в env картриджа. Это **новая
связь консоль→ядро→картридж** (Закон 3) → свой контракт, отдельный ADR (подпись Куратора F5, QUEUE
2026-07-18). Отдельный от дозора (ADR-0018): дозор = что скаут СМОТРИТ (капитализация/оборот/возраст),
Динамика = что бот БЕРЁТ из печки (движко-скоуп). ADR-0018 п.3 запретил движко-скоуп на канале дозора —
здесь зеркально: дозорный скоуп на канал Динамики не тащим.

## Решение (одобрено Куратором, подпись QUEUE 2026-07-18)
Зеркало дозора (ADR-0018), но своим каналом и со своим механизмом применения:
- **Таблица `dynamic_settings`** (миграция `0014`, зеркало `scout_settings`): `instance_id` PK → `desired`
  JSONB + `updated_at/updated_by`. Ядро = ИСТИНА. Аддитивна (down_revision `0013_scout_settings`).
- **Критерии = движко-скоуп:** `min_score` (доп-порог скора ПОВЕРХ дозорного), `stack_max` (предохранитель
  стека), `fresh_bars` (свежесть ≤ N баров, 0=выкл). Pydantic-границы на все три; у `stack_max` **верхний
  кап обязателен** (снять предохранитель через PUT нельзя: `stack_max=100000` → 422).
- **Routes (`routes_dynamic.py`, зеркало `routes_dozor.py`):** PUT `/instances/{id}/dynamic/settings`
  (operator-RBAC + `audit_log` before/after; портал отказ — маршрут только operator, Закон 5), GET
  оператору (desired поверх дефолтов), GET `/dynamic/settings/self` (instance-токен — картридж читает своё).
- **Дефолты канала = ЗЕРКАЛО генных дефолтов картриджа** (`min_score=0/stack_max=10/fresh_bars=0`): ядро
  молчит/недоступно ≡ ядро с дефолтами — одно поведение (нет строки → отдаём генные дефолты).
- **Команды `dynamic_apply` НЕТ** (в отличие от дозора) — живое применение обеспечивает re-fetch (ниже, D1).

## D1 — механизм применения: JSON-файл + живое чтение (ОСОЗНАННОЕ расхождение с дозором)
Дозор source'ит shell-файл `SCOUT_*` и рестартит СКАУТ — потому что скаут = отдельный супервизорный
процесс (`start.sh:scout_supervise`). Провайдер же живёт **В адаптере-foreground** (`exec python -m
app.main`, PID 1) — рестартить его как скаут нельзя (рестарт адаптера = смерть контейнера со всем: движок,
скаут). Поэтому:
- Картридж (`dynamic_overrides.py`, зеркало `scout_overrides.py`): фоновая нить **ПЕРИОДИЧЕСКИ** (re-fetch
  ~5 мин, D1-доп.1 Куратора) тянет `/dynamic/settings/self` → пишет **JSON-файл критериев** атомарно
  (tmp+os.replace) ТОЛЬКО при изменении + лог смены («критерии динамики: min_score 35→50» — канал виден глазами).
- Провайдер (`dynamic_universe.py`) читает файл-критерии **ЖИВЬЁМ в начале каждого `_recompute`** (fallback
  = генные дефолты `cfg`). Применение БЕЗ рестарта. Инвариант дозора сохранён («ядро=истина, картридж
  забирает своё»), но foreground-провайдер применяет иначе.
- **Периодический re-fetch убирает нужду в команде `dynamic_apply`** (D2): правка в консоли доезжает до
  ЖИВОГО Борса за минуты сама. Мгновенное применение — техдолг, не Веха 1.
- **Анти-инъекция (страж 2 дозора):** в файл пишем ТОЛЬКО whitelist движко-ключи, значения строго int
  (сырую строку из сети не пишем). Дозорный/риск-скоуп физически отсутствует в whitelist (D3, разделение скоупов).

## Политика капа на живое сжатие (EDIT 2 Куратора)
`stack_max` меняется на бегу (D1-живьём). При СЖАТИИ (Оператор 10→5 при стеке 7) — **естественное убытие**:
принудительно НЕ выгоняем (выгон = рестарт-чёрн против анти-thrash Слоя 2), новых не добираем, пока стек не
спустится ниже капа. Карточка Борса показывает ЧЕСТНО «в стеке 7 · кап 5» (перебор подсвечен медью), без
вранья. Кап читается живьём каждый скан → `view()` честен. (Тест `test_cap_shrink_no_evict`.)

## Консоль (раздел 5 «Динамика»)
`Constructor.tsx::Universe()` режим «Динамика» оживлён: селектор инстанса (Борс) + 3 поля editable
(min_score/stack_max/fresh_bars) + «Сохранить»→ядро. У «Сохранить» честная подпись «применится в течение
~N минут» (re-fetch, не мгновенно). Подсказка у `min_score`: работает ПОВЕРХ дозорного порога печки (ниже
него эффекта нет — не поломка). «Капитализация/Топ-N» — disabled (дозор-скоуп, крутится на Разведке).
`BotCard.tsx` секция «Стек · рабочая вселенная» (символ/стадия/скор) + чип «в стеке k · кап N» — рендер
ТОЛЬКО если `engine_state.stack` есть (Персиваль/Галахад/фикс-набор — секции нет). Тема «слаш», 0 хардкод-цветов.

## Ремонт после аудита (5 линз, 2026-07-18)
Адверсариальный аудит Слоёв 1–3 нашёл 🔴 + дешёвые 🟡, закрыты сразу:
- 🔴 **`findings_for_universe` читал `f.get("state")`**, а вендор `build_scout` кладёт стадию под `status`
  (viewmodel:542) → стек Борса не наполнялся никогда, фича немая в проде. Фикс: читать `status`, маппить
  в контрактный `state` (как рабочий `mapper.scout_snapshot`). **Вендор-интеграционный тест** (реальная
  `storage.db`→`build_scout`→провайдер, не мок — урок S7) — падает без фикса, ловит регресс.
- 🟡 **Предохранитель `stack_max` — теперь и в коде картриджа** (`_load_criteria` кламп `min(100)`): env-путь
  `DYNAMIC_STACK_MAX` шёл мимо валидации ядра. «Страж в коде, не по договорённости» (принцип ADR-0018).
- 🟡 **`.gen` пишется атомарно** (tmp+os.replace): `start.sh` cat'ит его → иначе рваное чтение → рестарт.
- 🟡 **re-fetch не клоббрит критерии пустым** (кривой 200 без `settings` → файл не трогаем).
- 🟡 **`fresh_bars` не режет `forming`** (нет `bars_since_anchor` → свежесть не определена → пропускаем).
- Отложено в техдолг (Веха 2): пин символа с позицией (часть «б» гейта) + код-триппвайр LIVE_TRADING=1;
  эскалация `TERM→KILL` в супервизоре движка; sync-гвоздь `DYNAMIC_DEFAULTS` ядро↔картридж.

## Последствия
+ Оператор задаёт критерии отбора из консоли; ядро — durable-истина; бот подхватывает сам за минуты без
  рестарта и без команды. + Скоупы дозор/движок разделены (два канала, аудит проще). − Второй канал настроек
  (после дозора) — но зеркальный, дисциплина одна. − Применение не мгновенно (минуты re-fetch) — приемлемо
  для Вехи 1 (dry-run). Гейт-позиций/pin символа Вехи 2 — в ADR-0019 (двухчастный). DEMO: `LIVE_TRADING_ENABLED=0`.
