# ADR-0019 — Разъём внешней вселенной монет в геноме (S8 «Динамо-близнец»)

## Контекст
Список торгуемых монот `COINS_CONFIG` прибит гвоздями в `bots/pifagor/vendor/config/strategy.py`
(хардкод-словарь, env-переопределения нет). Бот-«динамик» (Борс) должен брать вселенную из живой
разведки («печки»), а не из статики — это дыра образца, а не поведение, которое надо сохранять.
ADR-0018 п.3 уже указал путь: «Движок/риск (…COINS_CONFIG…) каналом дозора не ходят НИКОГДА — их путь
профили Конструктора (отдельная веха)». Эта веха настала. Решение Оператора+Куратора — **Путь Б**: не
обходить образец обёрткой, а **починить дыру** — сделать список монот внешним разъёмом. Умение идёт в
**геном** (закон эталона ADR-0014): каждый будущий бот рождается с «разъёмом под любой список», тумблер выкл.

## Решение (одобрено Куратором, подпись QUEUE 2026-07-18)
Аддитивный разъём в `strategy.py`: `COINS_CONFIG = _load_coins_config()`.
- **Источник — файл, не env-JSON.** `COINS_CONFIG_PATH` (env) = путь к JSON-файлу, который провайдер
  картриджа пишет атомарно (tmp+os.replace). НЕ инлайн-JSON в shell-source: символы из сети (Bybit) с
  shell-метасимволами исполнялись бы при старте — дыра, которую закрыл страж ADR-0018 п.2; + shell-квотинг
  JSON ломается (живой баг S7 «применил, разницы нет»). Файл убирает поверхность инъекции целиком.
- **REPLACE.** Внешний файл заменяет вселенную ЦЕЛИКОМ (не merge): либо внешний список, либо дефолт.
- **Дефолт байт-в-байт.** Env не задан / файла нет (холодный старт) / кривой вход → `_DEFAULT_COINS_CONFIG`
  (ТОТ ЖЕ объект). Персиваль/Галахад/paper разъёма не замечают — доказано тестом «монет=16».
- **Fail-loud.** Кривой JSON / не-объект / после нормализации 0 монет → проблема в `_env.ENV_ERRORS`,
  `config.validate()` останавливает старт понятно (дисциплина `env_int/env_float`), не сырым трейсбеком.
- **Нормализация.** Символы — regex `^[A-Z0-9]{2,15}USDT$`, верхний регистр (=де-дуп), мусор пропуск+лог.
  Ключ `leverage` гарантированно присутствует (main.py:234 берёт прямым subscript). Диапазоны — `validate`.
- **Скоуп — движок + адаптер** (оба отражают торгуемую вселенную). Скаут И скринер запускаются с
  `env -u COINS_CONFIG_PATH` → остаются на дефолтной вселенной (`scout/bars.py` классифицирует бары по ней).
- **Секретов канал не несёт** — только монеты; в лог движка попадает лишь `len(symbols)`.

## Амендмент Закона 6 + ADR-0016 (это — суть подписи)
Закон 6 (CLAUDE.md): «Движок Пифагора — отдельный репозиторий (pifagor-v81). Здесь его код не менять и не
копировать». ADR-0016: «vendor неприкосновенен» / «0 строк в vendor/». Правка `strategy.py` — первая
правка вендоренного субсета за всю дугу. Примиряем ЯВНО, три уровня «эталона» — разные сущности:
1. **Репозиторий `pifagor-v81`** (Закон 6) — НЕ трогаем никогда (тег-указатель `malysh-merlin/v8.3-b75bd17`).
2. **Замороженный архив b75bd17** (ADR-0014, tar.gz + SHA256) — disaster-recovery эталон, НЕизменен.
3. **Живой геном** `bots/pifagor/vendor/` — рантайм-субсет картриджа. ЕГО можно эволюционировать
   санкционированными дельтами; интеграция всё равно контрактная (env-файл), поведение по умолчанию — эталон.

**Инвариант живого генома (дословно):** «Замороженный архив b75bd17 неизменен. Живой геном = архив +
ТОЛЬКО санкционированные дельты. Каждая дельта = отдельный подписанный ADR. Реестр дельт ведётся в ADR-0019.»

**Реестр санкционированных дельт живого генома:**
1. `config/strategy.py` — разъём `COINS_CONFIG_PATH` (ADR-0019, S8). *Пока единственная.*

## Страж-дрейфа (CI — УСЛОВИЕ подписи, не пожелание)
`bots/pifagor-cartridge/tests/genome_manifest.json` пиннит SHA256 всех 69 файлов vendor: 68 frozen
(== b75bd17, git-verified: против main изменён только `strategy.py`) + 1 sanctioned (`strategy.py`).
`test_genome_drift_guard.py` сверяет дерево: любая НЕсанкционированная правка/добавление/удаление файла
vendor ИЛИ вторая дельта без ADR → **красный CI**. Граница «живой геном vs эталон» становится аудируемой,
а не «на слово». Вторая дельта = осознанно: новый ADR + регенерация манифеста + правка ожидания в тесте.

## Гейты Вехи 2 — ДВУХЧАСТНЫЙ гейт (уточнено досмотром Куратора, Слой 3)
**Гейт Вехи 2 состоит из ДВУХ обязательных частей (одной мало — досмотр S8/Слой 3 показал).**
Первопричина: сетап, дошедший до входа, РАЗРЕШАЕТСЯ у обходчика импульсов (`state='committed'` из
`scan.py:43` — внутреннее разрешение, в находки НЕ эмитится) → исчезает из печки → `missed`→`exit_scans` →
провайдер выкинул бы символ из набора через пару сканов ПОСЛЕ входа. Тогда одного «не рестартить при
позициях» недостаточно: рестарт лишь отложится, а затем движок всё равно поднимется БЕЗ символа, под
которым открыта позиция.
- **(а) Гейт-позиций рестарта.** Провайдер меняет вселенную → мягкий рестарт ТОЛЬКО движка. Веха 1
  (dry-run): позиций нет, гейт — заглушка (no-op в `start.sh:engine_supervise`). Веха 2: НЕ рестартить
  движок, пока есть открытые позиции/ордера.
- **(б) Пришпиливание символа (pin поверх `exit_scans`).** Символ с живой позицией/ордером ПРИШПИЛЕН в
  наборе: провайдер НЕ выкидывает его по `exit_scans`/исчезновению из печки, пока по символу есть живая
  позиция/ордер (полный финал — определение Оператора: всё закрылось / всё отменилось). Так движок после
  рестарта видит символ, под которым позиция.
- **Веха 2 (демо-ордера) НЕ открывается без ОБЕИХ частей** (а)+(б). Веха 1 (dry-run): слот свободен по
  стадии/TTL (сетап ушёл из активных / протух); обе части — задел (позиций нет, оба no-op).
- **DEMO/dry-run.** `LIVE_TRADING_ENABLED=0` у Борса до отдельного гейта Вехи 2 (Оператор+Куратор явно).

## Последствия
+ Список монот больше не прибит гвоздями; динамика-умение — в геноме, тумблер выкл (Персиваль/Галахад/paper
  не задеты дефолтом). − Живой геном официально отходит от «0 vendor» — но под аудируемым стражем и реестром
  ADR. Смена вселенной требует рестарта движка (не hot-reload) — приемлемо (гистерезис против дребезга, S8 Слой 2).
Критерии Динамики (скор/свежесть/кап/цель) — ОТДЕЛЬНЫЙ канал ядра, **ADR-0020** (не этот; scout-whitelist не расширяем).
