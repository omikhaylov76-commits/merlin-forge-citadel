# ADR-0003. Railway для v1, оркестрация через драйвер
Контекст: Оператор уже живёт на Railway; нужен программный деплой инстансов.
Решение: оркестратор работает через интерфейс InfraDriver. v1 — RailwayDriver (GraphQL API:
serviceCreate из образа + variables, redeploy, delete). Второй драйвер DockerDriver (VPS) —
заготовка интерфейса, реализация при необходимости.
Факты (проверено 2026-07): статический исходящий IP — Pro-план, IP общие (шарятся между
клиентами Railway), исходящий только; цены Pro $20/мес + RAM $10/GB/мес + CPU $20/vCPU/мес.
Оценка флота 12 ботов + сервисы ≈ $70–110/мес. Регион ботов — Southeast Asia (ближе к Bybit).
Последствия: ноль своей инфры на старте; переезд на VPS = реализация DockerDriver, без ломки.
Риск принят: IP-whitelist на общем IP слабее выделенного; компенсация — права ключей
«только торговля» + мониторинг. Пересмотр при выходе клиентов на mainnet-объёмы.
