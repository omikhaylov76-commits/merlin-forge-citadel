# ADR-0021 — Разъём warm-ритма в геноме (S8 «Динамо-близнец»)

## Контекст
Движок подхватывает «тёплые» (`auto_eligible` нетронутый PENDING) сетапы ТОЛЬКО раз при старте
(`warm_start_auto`, `main.py:184`, гейт `WARM_ON_START`). В динамике (Борс) вселенная меняется внутри дня;
монета, ставшая `auto_eligible` ПОСЛЕ старта, никем не перепроверяется — висит непоставленной (F-lookahead,
доказано вживую: TLM-класс). Курсорный no-backfill живого цикла её тоже не берёт (сформировалась до входа).
Нужна КАДЕНЦИЯ уже существующего безопасного инварианта — не новая торговая логика.

## Решение (одобрено Куратором, подпись QUEUE 2026-07-22, ветка А «геном»)
Аддитивный разъём warm-ритма в геноме — ДВЕ каденции поверх существующего `warm_start_auto`:
1. **Самоход** — авто-warm (`auto_eligible`) на КАЖДОЙ границе SIGNAL_TF (не раз при старте). Врезка:
   `main.py:_maybe_4h_cycle` ПОСЛЕ `run_4h_cycle`. Гейт-флаг `WARM_EACH_CYCLE` (env, дефолт **OFF**) —
   проверяется ДО вызова (fail-closed): OFF → авто-warm НЕ зовётся вообще (путь флота не меняется ни на инструкцию).
2. **Горн** — on-demand: интент `WARM_AUTO_NOW` (`config_log`) ← адаптер по кнопке `scan_now` (существующая
   команда ядра/консоли, новую НЕ заводим) → движок консюмит (single-shot по in-memory курсору,
   `init = latest-id` на старте → рестарт не повторяет старый интент) → тот же авто-warm.
- **Ритм из настроек, не хардкод «4h».** Граница `_maybe_4h_cycle` считается от `SIGNAL_TF` (парсер
  `_signal_tf_ms`), не от константы `FOUR_HOUR_MS`. `_signal_tf_ms("4h") == FOUR_HOUR_MS` (14_400_000) —
  **ТОЖДЕСТВО**, доказано тестом → флот байт-в-байт. Задел под 1h без переписывания движка.
- **Только `auto_eligible`.** Общий проход `warm_auto_now` (вынесен из `warm_start_auto` — тело после гейта
  `WARM_ON_START`) ставит через `_warm_one` = ТОЛЬКО `auto_eligible` (нетронутый PENDING). Пере-якорь
  (F-warm-button) НЕ протаскиваем — `_warm_one_button`/`WARM_APPLY`/`maybe_warm` не трогаем.
- **Гейты сохранены.** `_warm_gate` (пауза/kill/equity) + `_warm_cap_ok` (кап-стек 10) + `has_active`
  (не задваиваем, открытую позицию не трогаем — урок 18.07) — переиспользуются, не дублируем.
- **Скоуп — движок.** Адаптер (0-vendor) лишь пишет интент горна в БД воркера (как `pause`/`stop_close`).
  Скаут/скринер не задеты. Фильтр входа (4h-only/tracking-ready/`auto_eligible`/кап10, ADR-0019) уже принят.

## Амендмент Закона 6 — ВТОРАЯ санкционированная дельта (реестр в ADR-0019)
Вторая правка живого генома (первая — разъём монот, ADR-0019). Инвариант живого генома (ADR-0019) в силе:
архив b75bd17 неизменен; живой геном = архив + ТОЛЬКО санкционированные дельты; каждая = свой ADR.
**Файлы дельты (frozen→sanctioned в `genome_manifest.json`, sha256+adr):** `app/main.py`, `app/cycle.py`.
`test_genome_drift_guard.py`: ожидание = ровно {`config/strategy.py`→0019, `app/main.py`→0021,
`app/cycle.py`→0021}; иная правка/файл vendor без ADR → красный CI (условие подписи, как для 0019).

## Условия подписи Куратора (обязательны, оба покрыты тестами)
1. **Тождество 4h — тестом:** `_signal_tf_ms("4h") == FOUR_HOUR_MS` точным равенством + последовательность
   границ `_maybe_4h_cycle` на серии now_ms идентична дореформенной. «Страж Персиваля в коде».
2. **Гейт-флаг fail-closed ДО авто-warm:** флаг OFF → авто-warm не вызывается ВООБЩЕ (тест «не вызван»).

## Последствия
+ Динамик-бот подхватывает свежие `auto_eligible` сетапы вовремя (кнопкой-горном + периодически-самоходом),
  не завися от случайного рестарта на смене вселенной. Ритм следует SIGNAL_TF. − Живой геном отходит от
  «0 vendor» второй дельтой — под аудируемым стражем + реестром ADR. Флот (флаг OFF + тождество 4h) байт-в-байт.
- **Горн — in-memory single-shot** (`init = latest-id` на старте), НЕ персистентный ack: намеренно НЕ заводим
  колонку в `storage/db.py` (минимизируем вендор-поверхность до 2 файлов). Рестарт с зависшим интентом = ≤1
  повтор, безопасно (`has_active`-skip). Осознанный размен поверхности на строгость персистентности — часть ветки А.
