# Durable engine-БД картриджа (persistent volume) — #57

## Зачем
engine-БД картриджа по умолчанию эфемерная (локальный SQLite стирается на каждом передеплое). Для
боевого бота это дырка: на КАЖДОМ рестарте теряется HWM (пик equity), трек сетапов, компаунд
working-капитала, память kill-switch. Плюс это КОРЕНЬ ложной защёлки kill-switch при ручной смене
баланса: свежая БД + кривой ранний замер equity читаются как просадка → защёлка (см. #Персиваль-ks).

## Эталон (в исходнике — новая копия рождается с этим)
`bots/pifagor-cartridge/start.sh` (main, коммит `ac44374`):
`export DB_PATH="${DB_PATH:-/data/pifagor.db}"` + `mkdir -p /data`.
→ engine-БД ВСЕГДА на монтируемом `/data/pifagor.db`. Без тома — эфемерно на /data (как раньше, иной
путь). Скаут свой `DB_PATH` задаёт сам (scout.db), не задет. Адаптер читает ту же `config.ops.DB_PATH`.
⚠️ НЕ монтировать том в `/pifagor/*` — там вендор-код образа, том перекрыл бы его.

## Рецепт: сделать конкретного бота durable (per-instance инфра — том)
1. **Бот ПЛОСКИЙ?** Нет открытой позиции/живых ордеров (перемиг на плоском боте — норма).
2. **Том Railway** на сервисе бота, mount path `/data`:
   - Дашборд: сервис → Add Volume → `/data`; ИЛИ
   - API: `volumeCreate(input:{projectId, environmentId, serviceId, mountPath:"/data"})`.
   Создание тома триггерит передеплой → engine-БД встаёт на `/data/pifagor.db`.
3. **Первый durable-старт с ре-базой** (peak_equity → текущий баланс, а не унаследованный пик):
   env `MF_RISK_REBASELINE_ONCE=1` (+ `MF_RISK_REBASELINE_TO=<баланс>`) → передеплой.
   ⚠️ **ПОТОМ ФЛАГ СНЯТЬ** (`variableDelete`) — иначе `risk_rebaseline.py` сбрасывает HWM на КАЖДОМ
   рестарте, стирая рост. На свежем томе peak и так встаёт на баланс (флаг — страховка); критичен он,
   если в томе уже есть БД с кривым пиком.
4. **Проверка:** kill-switch=0; на тестовом передеплое в логах `engine-БД: DB_PATH=/data/pifagor.db`,
   ре-база НЕ бежит (флаг снят), warning «backfill таймаута ПРОПУЩЕН (свежая БД)» ОТСУТСТВУЕТ
   (= БД пережила передеплой).

## Тонкости
- Дефолт `/data` действует на ВСЕ картриджи на их след. передеплое (путь БД сменится → свежая БД). Для
  эфемерных (dry-run) ботов это норма — они и так теряют состояние. Durable = только там, где есть том.
- Откат: удалить том Railway (вернётся эфемерность). Свежего снимка «до» нет — том создавался пустым.
- Смежное: штатная кнопка сброса kill-switch (`clear_killswitch`) НЕ трогает HWM → при смене баланса не
  спасает; нужна ре-база (`risk_rebaseline.py` или будущая аудируемая команда `REBASELINE_RISK`, #4).
